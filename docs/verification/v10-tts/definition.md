# V10: TTS品質・コスト

## 問い

動的模範解答のTTS生成について、品質・コスト・速度は学習用途として許容範囲か?

## 背景

フェーズCでは動的模範解答のTTS音声を使ってリスニング・オーバーラッピング練習を行う。TTS音声の品質はフェーズCの学習効果に直結する。

TTS生成はフェーズB中にバックグラウンドで開始する（11-latency-strategy.md）。

不確実な点:
- ElevenLabs等のTTS品質が学習用途に十分か（自然なイントネーション、速度調整の品質）
- 動的に生成された英文（文法的に完璧でない場合がある）をTTSに通した時の品質
- 1パッセージ（約100語）のTTS生成速度
- 1日あたりのTTSコスト（複数パッセージ × 動的模範解答）
- 英語バリエーション（American / British）の対応品質

## 検証方法

1. ElevenLabsの複数音声（American / British）で同じテキストを生成
2. ���準的な英文と、やや不自然な英文（動的模範解答を模擬）の両方を生成
3. 生成速度を計測
4. 0.75x / 1.0x / 1.25x の速度調整で品質を確認
5. コストを概算

## 合格基準

- 1パッセージ（100語）のTTS生成が10秒以内
- 音声品質がリスニング練習に使えるレベル（不自然さが学習の妨げにならない）
- American / British の区別が明確
- 1日あたりのコストが$1以下（1ユーザー、10パッセージ程度の想定）

## フォールバック

- 動的模範解答ではなく事前模範解答のTTSのみ使用（コスト削減 + 品質安定化）
- TTS品質が不十分なら、フェーズCのリスニング練習をテキスト表示のみにする

## 解決: ElevenLabsの品質を確認済み（2026-04-06）

開発者が事前にElevenLabsのTTS品質を確認済み。学習用途として十分な品質であることを確認したため、本検証は不要（resolved）とする。
