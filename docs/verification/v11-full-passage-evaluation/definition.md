# V11: 本番発話の内容評価

## 問い

フェーズCの本番発話で、約60秒のパッセージ全体の音声を評価し、合否判定 + 差分情報を実用的な速度で返せるか?

## 背景

フェーズCの本番発話では、日本語のお題のみ表示した状態でパッセージ全体を英語で発話する。評価は2段階:

1. **時間判定**（即時）: ターゲットWPM内に収まったか。クライアント側で即時計算
2. **内容評価**（LLM #8）: 全センテンスが英語で発話されているか、動的模範解答との差分は何か

不確実な点:
- 60秒程度の音声をGeminiに投げた時の応答時間
- 長い音声の途中でノイズや沈黙がある場合の処理
- 合否判定の精度（特に「英語にできなかった箇所がある」の検知）
- 差分要約の有用性（何が違っていたかを学習者が理解できる形で出力できるか）

## 検証方法

1. 約60秒のパッセージ音声を複数パターンで録音:
   - 全センテンス英語で発話（合格ケース）
   - 一部が日本語のまま（不合格ケース）
   - 英語だが意味が大きく異なる（判断が難しいケース）
2. LLM #8 相当のプロンプトで評価を実行
3. 応答時間と判定精度を計測

## 合格基準

- 応答時間: 中央値5秒以内
- 合否判定の正答率: 85%以上
- 差分要約が学習者にとって有用（何が違ったか理解できる）

## フォールバック

- 時間内完了のみ即時判定、内容���価は遅延表示（セッション後に確認可能）
- パッセージを分割して評価（センテンスごとに送信）
- 内容評価を省略し、合否は時間判定のみとする
