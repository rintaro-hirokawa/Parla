# V1: パッセージ生成品質

## 問い

多様なソーステキストから、CEFR準拠の学習用パッセージを安定して生成できるか?

## 背景

パッセージ生成（LLM #1）はアプリの入口。ここの品質が低いと、以降のすべてのフェーズ（フィードバック、SRS、通し練習）の品質が連鎖的に劣化する。

不確実な点:
- ソースの種類（短文記事、長文記事、日本語記事、YouTube要約、技術文書）によって出力品質がどれだけ変動するか
- LLMが「最適なパッセージ数・分量を決定する」（04-source-management.md）設計で、実用的な分割ができるか
- CEFR レベルの制御が効くか（B1指定で本当にB1相当の難易度になるか）
- ヒント（2段階）が段階的な情報開示として機能するか
- パッセージ形式（説明型/対話型/意見型）の指定が意図通り反映されるか

## 検証方法

### 検証パターン（1パターンに絞る）

最もリスクが高い組み合わせで検証する:

- **ソース**: YouTube動画の要約テキスト（口語的・構造が緩く、LLMにとって最も難しい入力）
- **CEFRレベル**: B1（レベル制御が効いているか確認しやすい）
- **英語バリエーション**: American English
- **パッセージタイプ**: 説明型

### 評価観点

各出力について以下を確認する:

| 観点 | 確認内容 |
|------|---------|
| パッセージ分割 | 1パッセージが約100語、5〜8文で構成されているか |
| 日本語プロンプト | 自然な日本語か。英語の直訳になっていないか |
| 模範英文 | 指定CEFRレベルに適合しているか（B1なのにC1語彙が多用されていないか等） |
| ヒント | 2段階で段階的に情報が増えているか（詳細は 07-hint-system.md 参照） |
| 形式の反映 | 説明型/対話型/意見型の指定が出力に反映されているか |
| ソース忠実性 | ソースの内容・トピックを適切に反映しているか |

## 合格基準

- 生成されたパッセージが「そのまま学習に使える」品質
- CEFRレベルの明らかな逸脱がない（B1指定でC1相当の文が混在しない）
- ヒントが段階的情報開示として機能している
- 日本語プロンプトが自然で意味が取れる

## フォールバック

- 入力ソースの制約を厳しくする（英語記事のみ、一定文字数範囲等）
- パッセージ数・分量をLLM判断ではなく固定ルールにする
- ヒント生成をパッセージ生成と別コールに分離して品質を上げる
