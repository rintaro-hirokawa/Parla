# V5: リトライ判定の速度と精度

## 問い

フェーズBの即時リトライで、1センテンスの発話音声を「正解/不正解 + 一言コメント」として3秒以内に判定でき、かつその判定は妥当か?

## 背景

フェーズBでは、日本語のみ表示してユーザーが再発話する即時リトライを最大3回行う。このフィードバックループが遅いとテンポが崩れ、学習体験が大幅に劣化する。

LLM #5 は出力トークンを最小限にする設計（正解/不正解 + 一言コメント）。

不確実な点:
- 音声入力 → 正誤判定の応答時間が3秒以内に収まるか
- 正誤判定の精度（模範解答と完全一致でなくても「意味的に正しい」ものを正解とすべき）
- 一言コメントが学習者にとって有用か

## 検証方法

### テスト音声の準備

1. **正解**: 模範解答とほぼ同じ発話
2. **表現は異なるが意味的に正解**: 別の言い回しだが意味は同じ
3. **部分的に正解**: 前半は正しいが後半が崩れた
4. **不正解**: 日本語が混ざる、意味が異なる

### 検証手順

1. 各テスト音声に対してLLM #5を実行
2. 応答時間を計測
3. 正誤判定と一言コメントの妥当性を人間が評価

## 合格基準

- 応答時間: 中央値3秒以内、最大値5秒以内
- 正誤判定の正答率: 85%以上
- 「表現は異なるが意味的に正解」を正解と判定できる率: 80%以上

## フォールバック

- STTベースのテキスト比較（音声→テキスト変換後に文字列類似度で判定）
- 正誤判定のみ即時（コメントは省略してレイテンシを削減）
- Gemini Flash に固定
