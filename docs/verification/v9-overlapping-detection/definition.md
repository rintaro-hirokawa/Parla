# V9: オーバーラッピング遅れ検知

## 問い

フェーズCのオーバーラッピング練習で、ユーザー発話と模範TTS音声のズレをフレーズレベルで検知し、遅れた箇所と原因を指摘できるか?

## 背景

オーバーラッピングでは、模範TTS音声と同時にユーザーが発話する。遅れやズレが発生した箇所を検知・指摘することで、どのフレーズが言いにくいかをフィードバックする。

LLM #7 はユーザー発話音声 + 模範TTS音声 + 模範英文テキストを入力として受け取る。

### 技術アプローチ

ElevenLabs Forced Alignment API と LLM の2段構成で検知する。

**Forced Alignment（遅れ箇所の定量検出）:**
- 模範TTS音声・ユーザー発話音声それぞれに、同じ模範英文テキストを与えてForced Alignmentを実行
- 単語レベルのタイムスタンプを取得し、両者の差分からフレーズごとの遅れ（秒数）を算出
- 各単語の `loss` 値でアライメント信頼度を確認（言えなかった・言い間違えた箇所はlossが高くなると期待）
- 模範TTS側のタイムスタンプは事前計算してキャッシュ可能 → リアルタイムではユーザー発話側の1回のAPI呼び出しのみ

**LLM（遅れ原因の推定）:**
- Forced Alignmentで検出された遅れ箇所の情報 + 模範英文テキストをLLMに渡す
- 遅れの原因を推定（発音困難、語彙想起の遅れ、構文組み立ての迷い等）
- 学習者向けのフィードバック文を生成

```
模範TTS音声 ──→ Forced Alignment ──→ 模範タイムスタンプ（キャッシュ可）
                                        ↓ 比較
ユーザー発話 ──→ Forced Alignment ──→ ユーザータイムスタンプ
                                        ↓
                                   フレーズ遅れ検出（定量値）
                                        ↓
                                   LLMで原因推定 + フィードバック生成
```

### 不確実な点

- ユーザーが模範と異なる表現で発話した場合のForced Alignmentの挙動（loss値の変化パターン）
- 60秒程度の音声に対するForced Alignment APIのレイテンシ（公式未公開、数秒以内と推測）
- 遅れの閾値設定（何秒のズレから「遅れ」と判定するか）
- LLMによる原因推定の妥当性

### 前提条件

- ユーザー発話のみマイクで録音する（模範TTS音声はスピーカーから再生し、録音に混入させない）

## 検証方法

### Step 1: Forced Alignment の精度検証

1. 模範TTS音声を用意し、Forced Alignmentで単語タイムスタンプを取得（模範基準値）
2. 以下のパターンでユーザー発話を録音し、それぞれForced Alignmentを実行:
   - 模範と同じ速度で発話（ズレなしの基準ケース）
   - 意図的に特定フレーズを遅らせた発話
   - 一部を言い間違えた・言えなかった発話
3. 両者のタイムスタンプ差分を計算し、遅れ箇所が正しく検出されるか確認
4. loss値が言い間違え・言えなかった箇所で有意に上昇するか確認

### Step 2: LLM 原因推定の検証

1. Step 1で得た遅れ箇所データ + 模範テキストをLLMに入力
2. 原因推定の妥当性を人手で評価
3. フィードバック文の有用性を確認

### Step 3: レイテンシ計測

1. 60秒程度の音声でForced Alignment APIの応答時間を計測（10回以上）
2. LLM原因推定の応答時間を計測
3. 合計レイテンシがUX要件を満たすか確認

## 合格基準

- 意図的に遅らせた箇所の70%以上を正しく検知
- 遅れていない箇所を遅れと誤検知する率が20%未満
- Forced Alignment APIの応答時間: 中央値3秒以内（60秒音声）
- LLM原因推定を含む全体レイテンシ: 中央値8秒以内

## フォールバック

- LLM原因推定を省略し、Forced Alignmentの定量値のみでフィードバック（「このフレーズで0.8秒遅れました」）
- Forced Alignment APIのレイテンシが許容範囲外の場合、全体の所要時間比較のみ（パッセージ全体で模範より何秒遅いか）
- フレーズレベル検知は将来拡張として保留
