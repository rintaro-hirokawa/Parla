# V3: 意味的重複検出

## 問い

新規抽出された学習項目と既存ストック済み項目を意味的に照合し、重複と再出を正しく判定できるか?

## 背景

学習項目の照合は文字列一致ではなく、LLMによる意味的照合で行う（05-learning-items.md）。これは LLM #4 のフィードバック生成時に、既存ストック済み学習項目リストをコンテキストとして渡すことで実現する。

不確実な点:
- **偽陽性（過剰な重複判定）**: 実際には異なるパターンなのに同一と判定してしまう
  - 例: "more X than Y"（比較級）と "the most X"（最上級）を同一と判定
- **偽陰性（重複の見逃し）**: 意味的に同一のパターンなのに別物と判定してしまう
  - 例: "be responsible for ~ing" と "take responsibility for ~ing" の関係性を見逃す
- **ストック数が増えた時のスケーラビリティ**: 100個、500個とストック済み項目が増えた時にプロンプトが長くなり、照合精度が落ちないか
- **再出検知の正確性**: 過去に間違えた項目をまた間違えた時、それを「再出」と正しく認識できるか

## 検証方法

### テストセットの準備

以下の学習項目ペアを用意する:

**重複ペア（同一と判定すべき）:**
- "A is more X than B" と "A is much more X than B"（修飾語追加のみ）
- "be responsible for ~ing" と "be responsible for doing something"（表記揺れ）
- "environmental impact" と "impact on the environment"（語順変化）

**非重複ペア（別物と判定すべき）:**
- "more X than Y"（比較級）と "the most X"（最上級）
- "take a look at" と "take care of"（動詞句だが意味が異なる）
- "be used to ~ing"（慣れている）と "used to do"（以前は〜していた）

**再出ケース:**
- ストック済みの "be responsible for ~ing" が含まれるセンテンスで、ユーザーが同じ項目をまた間違える

### 検証手順

1. ストック済み学習項目リスト（10個、50個、100個の3パターン）を準備
2. テスト音声 + ストック済みリストをLLM #4に入力
3. 出力された新規/再出の判定を評価

### 評価観点

| 観点 | 確認内容 |
|------|---------|
| 重複検知 | 意味的に同一のパターンを同一と判定できるか |
| 非重複の区別 | 異なるパターンを別物として扱えるか |
| 再出検知 | ストック済み項目の再出を正しく「再出」とラベル付けできるか |
| スケーラビリティ | ストック数100個でも照合精度が維持されるか |

## 合格基準

- 重複ペアの正答率: 80%以上
- 非重複ペアの正答率: 90%以上（過剰な統合は学習に悪影響が大きいため厳しめ）
- 再出検知の正答率: 85%以上
- ストック数100個でも上記基準を維持

## フォールバック

- 意味的照合の前段でパターンの正規化処理（記号統一、語順正規化等）を入れる
- ストック数が多い場合、関連する大分類/サブタグでフィルタしてからLLMに渡す
- 照合に自信がない場合は「要確認」として保留し、ユーザー判断に委ねる
