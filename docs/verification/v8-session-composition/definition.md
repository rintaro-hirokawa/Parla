# V8: セッション構成のLLM判断品質

## 問い

LLMに翌日のセッションメニューを構成させた場合、決定論的なルールベーススケジューリングより有意に良い結果が出るか?

## 背景

セッション構成（LLM #2）はハイブリッド設計: SRSスケジューリングやデータ集約は決定論的ロジック、全体の配分比率はLLM判断（03-session-structure.md）。

LLMが判断する部分:
- ブロック1（レビュー）とブロック2（新規素材）のバランス
- 過去のパフォーマンスを踏まえた優先度調整
- モチベーションメッセージ

不確実な点:
- LLMの判断が「適切な学習計画」として人間が納得できるものか
- 入力データ（SRSキュー深度、パフォーマンス履歴、疲労パターン）をLLMが適切に解釈できるか
- 決定論的なルール（例: レビュー上限20問、バックログ時はレビュー優先）だけで十分ではないか

## 検証方��

### テストデータの準備

以下の学習状態シナリオを模擬する:

1. **通常状態**: レビュー15問、新規素材あり、パフォーマンス安定
2. **バックログ状態**: レビュー30問超、数日サボった後
3. **疲労兆候**: 直近3セッションでWPM低下傾向

### 検証手順

1. 各シナリオのデータをJSON形式で準備
2. LLM #2に投入してセッションプランを生成
3. 同じデータで決定論的ルールベースのプランも作��
4. 両者を比較評価

### 評価観点

| 観点 | 確認内容 |
|------|---------|
| バランス | レビューと新規のバランスが妥当か |
| 状況適応 | バックログ・疲労等の状況に適切に反応しているか |
| ルールベースとの差 | LLM判断がルールベースより明確に良い点があるか |

## 合格基準

- 3シナリオ中2つ以上で、ルールベースと同等以上のプランを生成
- 少なくとも1シナリオで、���ールベースでは対応できない文脈判断（例: 疲労時の配慮）が見られる

## フォールバック

- セッション構成を全てルールベースにする（LLM判断を使わない）
- LLMの役割をモチベーションメッセージ生成のみに限定する

## 解決: 決定論的アプローチを採用（2026-04-06）

検証実施前に、セッション構成を決定論的ルールベースに変更する方針を決定した。理由:

- LLMが判断すべき領域（配分比率、優先順位）が実質的に狭く、ルールで十分カバーできる
- 3パターン（復習+新規 / 復習のみ / 新規のみ）の切り替えルールで主要ケースを網羅できる
- ルールベースの方がデバッグしやすく、ユーザーへの説明も容易

本検証は不要（resolved）とする。
