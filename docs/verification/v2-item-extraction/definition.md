# V2: 学習項目抽出精度

## 問い

日英混在の発話音声から「なぜ英語で言えなかったか」を正確に推論し、適切な粒度の学習項目として抽出できるか?

## 背景

フィードバック生成（LLM #4）はアプリのコア価値。ユーザーが日本語プロンプトに対して発話した音声を分析し、以下を出力する:

1. 発話内容の再現（日本語部分は要約、英語部分はできるだけ拾う）
2. 動的模範解答（ユーザーの発話を活かした模範英文）
3. 学習項目リスト（新規/再出フラグ、confidence、タグ分類）

不確実な点:
- 日英混在音声をGeminiが正確に聞き取れるか（日本語で言った部分と英語で言った部分の判別）
- 「言えなかった原因」の推論精度（語彙不足 vs コロケーション不足 vs 構文不足の判別）
- 学習項目の粒度が適切か（"A is more X than B" のような具体パターンか、"比較級" のような抽象カテゴリか）
- confidence値が信頼できるか（高confidenceの項目は本当に正しいか）
- タグ分類（大分類: 文法/語彙/コロケーション/構文/表現、サブタグ）が妥当か

## 検証方法

### テスト音声の準備

開発者自身が以下のパターンで発話を録音する:

1. **部分的に日本語が混ざる発話**: 途中で英語が出ず日本語に切り替えたケース
2. **大幅に異なる発話**: 意味は伝わるが構文・語彙が模範と大きく異なるケース

### 検証手順

1. V1で生成したパッセージを使用する（V1の検証結果に依存）
2. 各テスト音声に対してLLM #4を実行
3. 出力された学習項目を人間が評価

### 評価観点

| 観点 | 確認内容 |
|------|---------|
| 発話再現 | 日本語/英語の判別が正しいか。英語部分が正確に拾えているか |
| 動的模範解答 | ユーザーの発話を活かしつつ、自然な英語になっているか |
| 項目の粒度 | 具体的パターンレベルか（"be responsible for ~ing" レベル） |
| 原因推論 | 語彙/コロケーション/構文の判別が妥当か |
| confidence | 高confidence項目が実際に正しく、低confidence項目が曖昧なケースか |
| タグ分類 | 大分類・サブタグが妥当か |
| 再出検知 | ストック済み項目の再出を正しく検知できるか（V3と関連） |

## 合格基準

- 発話再現の日英判別が90%以上正確
- 抽出された学習項目のうち、人間が「妥当」と判断するものが70%以上
- 学習項目の粒度が具体パターンレベル（抽象カテゴリではない）になっている率が80%以上
- confidence高の項目の正答率が80%以上

## フォールバック

- 粗い抽出 + ユーザー編集で補完する運用（05-learning-items.md の「要確認候補」を多めに出す）
- 学習項目抽出を別コールに分離し、発話再現/動的模範解答とは独立に処理する
- confidence閾値を上げて自動ストックを控えめにし、ユーザー確認を増やす
