# 学習項目の設計

---

## 粒度

学習項目は具体的パターンレベルで管理する。

- 良い例: "A is more X than B"、"environmental impact"、"be responsible for ~ing"
- 悪い例: 「比較級」「不定詞」のような抽象的な文法カテゴリ

ユーザーが実際に「出せなかった表現」をそのまま学習項目として扱う。

---

## 種類とタグ体系

### 大分類（必須、1つ選択）

固定リストから選択する。

| 大分類 | 説明 |
|--------|------|
| 文法 | 時制、比較級など文法的パターン |
| 語彙 | 単語レベルの知識 |
| コロケーション | 語と語の自然な結びつき |
| 構文 | 文の組み立て方のパターン |
| 表現 | 慣用表現、定型フレーズ |

### サブタグ

大分類ごとにサブタグを持つ。サブタグは固定リストとし、運用しつつ必要に応じて拡張する。

文法サブタグ: 時制 / 比較 / 関係詞 / 仮定法 / 受動態 / 不定詞 / 動名詞 / 分詞 / 助動詞 / 冠詞 / 前置詞 / 接続詞

語彙サブタグ（品詞）: 名詞 / 動詞 / 形容詞 / 副詞 / その他

### 分野タグ（任意、複数可）

- ソースから自動付与される
- 固定リストではなくAIが自動生成する（例: ビジネス、技術、DIY、医療、環境）
- 大分類・サブタグとは異なり、動的に増えていく

---

## AI推論による学習項目の特定

- ユーザーが何がわからなかったかをAI側で推論する
  - 単語自体を知らないのか
  - 知っているが適切なコロケーションとして出せなかったのか
  - 構文の組み立てができなかったのか
- 既存ストック済み学習項目との照合（再出検知）はLLMによる意味的照合で行う
  - 文字列の完全一致ではなく、意味的に同じ学習項目かどうかをLLMが判断する

## 抽出UXフロー

フェーズBで学習項目が提示された時、原則としてAIが自動で反映する。ユーザー操作は例外修正のためにのみ提供する。

### 操作一覧

| 操作 | 説明 |
|------|------|
| 自動ストック | 信頼度が高い項目は自動でストックに追加。タグはAIが自動付与 |
| 要確認候補 | 信頼度が低い項目は候補として保留し、その場の進行は止めない |
| [編集] | 表現・意味・タグを修正する |
| [今回は追加しない] | 自動追加された項目を取り消す |
| [学習項目を自分で追加] | AIが検出しなかった弱点をユーザーが手動登録 |

### 遷移制御

- 学習項目の未処理状態で進行を止めない
- 確認ダイアログは出さず、要確認候補は後でまとめて見直せるようにする

### 再出表示

- 既にストック済みの項目を再び間違えた場合、強調表示する
- 「再出」ラベルと初出ソース情報を付与する

---

## 紐付け情報

各学習項目は以下の情報を保持する。

- どのソースから抽出されたか（初出ソース）
- 運用練習の全履歴
  - 日時
  - 正解/不正解
  - WPM
  - タイマー消費率
- ヒント使用履歴

---

## 学習項目一覧タブ

### フィルタ

- 大分類
- サブタグ
- 分野タグ
- ステータス（学習中 / 習得済み）

上記を組み合わせて絞り込む。

### 検索

- テキスト検索に対応する

### 一覧表示項目

各項目について以下を表示する。

- パターン（学習項目の表現）
- タグ（大分類、サブタグ、分野タグ）
- SRS段階
- 次回復習予定
- 初出ソース

---

## 学習項目詳細画面

### 成長ストーリー

- 初出ソース
- 全練習履歴（日時、正解/不正解、WPM、タイマー消費率）
- WPM推移グラフ

### ステータス情報

- ステータス（学習中 / 習得済み）
- SRS段階
- 次回復習予定

### 例文

- 初出時の具体例を表示する

### オンデマンド練習（将来機能）

- 詳細画面から「今すぐ練習」ボタンで即座に練習を開始できる
