# パッセージ学習サイクル

パッセージ単位の学習は、フェーズA（発話）-> フェーズB（フィードバック+即時運用練習）-> フェーズC（通し練習）の順に進行する。

---

## フェーズA: 発話

### 画面表示

- センテンスごとにハイライト表示する。前後のセンテンスは薄く表示し、文脈確認に利用できるようにする
- 録音中はレベルメーター/波形をリアルタイム表示する

### タイマーと自動遷移

- ターゲットWPMに基づくタイマーが自動開始する
- ユーザーが発話し、タイマー終了で自動的に録音終了する
- 録音終了後、次のセンテンスへ自動遷移する
- パッセージ全体（全センテンス）をしゃべり終えてからフィードバックに進む

### フィードバックの並列生成

- フィードバック生成はバックグラウンドで並列実行する
- センテンスごとに録音完了時点で即時処理を開始する

### ストック済み学習項目のヒント表示

ストック済み学習項目がそのセンテンスで使える場合、ヒントボタンからオンデマンドで確認できる。自動表示はしない。

- ユーザーがヒントボタンを押すと、そのセンテンスに関連するストック済み学習項目を表示する
- 表示内容: 学習項目名（例: "test ~ oneself"）
- ヒントの利用はSRS信頼度乗数に影響する（[07-hint-system.md](07-hint-system.md) 参照）

---

## フェーズB: フィードバック+即時運用練習

センテンスごとに以下の手順を実施する。

### 1. ユーザーの発話内容の再現

- 日本語部分は要約する
- 英語部分はできるだけ拾う

### 2. 動的模範解答の提示

- ユーザーの発話をできるだけ活かした模範英文を生成・表示する

### 3. 学習項目の提示

- **新規（高信頼）**: 新たに検出された弱点を提示し、自動でストックに追加する
- **新規（要確認）**: 信頼度が低い候補は「要確認」として表示し、即時SRSには投入しない
- **再出**: 既にストック済みの学習項目を再び間違えた場合、強調表示し、ユーザー操作なしで再出として記録する

### 4. 学習項目の自動反映

- 高信頼の新規学習項目は、タグ付きで自動ストックする
- ユーザーは必要な場合に限り [編集] / [今回は追加しない] / [学習項目を自分で追加] を行える
- 要確認候補はセッション後にまとめて見直せるよう保留し、即時SRSには投入しない
- 学習フローを止める確認ダイアログは出さない

### 5. 即時リトライ

- 英語を伏せ、日本語のみ表示する
- ユーザーが再発話する
- 正解判定を行い、正解なら次のセンテンスのフィードバックへ進む
- 最大3回で次へ進む

### フェーズBの方針

- 文法解説は不要（学習項目として集約すれば十分）
- 発音記号・モデル音声はフェーズCで提示する
- 即時リトライではヒントの利用を制限する（詳細は [07-hint-system.md](07-hint-system.md) 参照）
- 動的模範解答のTTS生成をバックグラウンドで開始する（フェーズC用）

---

## フェーズC: 通し練習ワークスペース

フェーズCは、動的模範解答を使って追加練習を行う任意フェーズとする。リスニング、オーバーラッピング、本番発話を同一画面上で提供し、実施順序は固定しない。

学習項目が0個かつWPMがターゲット内であればスキップ推奨とする。ユーザーはそのまま次のパッセージへ進んでもよいし、必要に応じてフェーズCの各練習を任意に使ってもよい。

### リスニング

- 動的模範解答のTTS音声を再生する
- 英文テキスト+学習項目ハイライトを表示する
- 発音記号を全単語の下に小さく表示する（ON/OFF切替可能）
- 何度でもリピート可能
- 再生速度調整: 0.75x / 1.0x / 1.25x

### オーバーラッピング

- 動的模範解答のTTS音声と同時に発話する（テキスト見てOK）
- 遅れ・ズレを検知し、該当箇所を指摘する（[技術検証 V9](../verification/v9-overlapping-detection/definition.md)）
- 何度でもリトライ可能
- 任意のタイミングで本番発話またはパッセージ完了画面へ移れる

### 本番発話

- 日本語のお題のみ表示する（英文なし、模範音声なし）
- ターゲットWPMに基づくタイマーを適用する
- ヒントなし
- 全センテンスを英語で発話する必要があり、英語にできなかった箇所があれば不合格
- 合格時は「通し発話達成」として記録する
- 不合格時は差分要約を表示し、ユーザーはリスニング・オーバーラッピング・本番発話を任意に継続できる
- 不合格でも、その場で次のパッセージへ進むことは可能とする

### フェーズCの運用方針

- UI上は「ステップを進める」構造にせず、3種類の練習を同じワークスペースで切り替えて使えるようにする
- システムは推奨アクションを提示できるが、実際にどの練習を何回行うかはユーザーが任意に決める
- フェーズCは追加練習の場であり、セッション進行を強制的に止めるゲートにはしない

---

## パッセージ学習完了と通し発話達成

フェーズA/Bを終えた時点で、そのパッセージの学習はセッション進行上は完了とする。フェーズC本番発話で、日本語のお題だけを見てターゲットWPM内で全センテンスを英語で発話できた場合は、追加で「通し発話達成」を記録する。

### ステータス定義

- **パッセージ学習完了**: フェーズA/Bが完了し、次のパッセージまたは次ブロックへ進める状態
- **通し発話達成**: フェーズC本番発話で合格した状態
- **通し発話未達成**: フェーズCを未実施、または本番発話で未合格のまま次へ進んだ状態

### 制限時間の自動算出

CEFRレベル -> ターゲットWPM -> パッセージのワード数に応じた制限時間を自動算出する（バッファを加算）。

### ターゲットWPMの目安テーブル

| CEFRレベル | ターゲットWPM |
|-----------|-------------|
| A2 | 90-100 |
| B1 | 110-130 |
| B2 | 140-170 |
| C1 | 180-200 |

---

## 模範解答の二層構造

### 事前模範解答

- パッセージ生成時にCEFRレベル準拠で作成する
- ヒント生成のベースとして利用する

### 動的模範解答

- フェーズBでユーザーの実際の発話を活かして生成する
- フェーズBでの表示およびフェーズCで使用する

---

## パッセージの形式バリエーション

| 形式 | 概要 |
|------|------|
| 説明型 | 事実を整理して伝える |
| 対話型 | 相手を想定したやりとり |
| 意見型 | 自分の考えを構成して伝える |

いずれの型でも、ユーザーが英訳すべき日本語は確定済みで提示する。
