# ヒントシステム

---

## 2段階のヒント

ヒントは段階的に情報量が増える2段階構成とする。

| 段階 | 名称 | 内容 | 例 |
|------|------|------|-----|
| 1 | ヒント1 | はじめの1語 + 学習者が出てきにくい単語 | "Chairman ... test / himself" |
| 2 | ヒント2 | 構文スケルトン + 学習者が出てきにくい単語 | "Chairman + 動詞(現在形) + 目的語 + himself / test" |

---

## 事前生成方式

- ヒントは模範解答ベースで事前生成する
- パッセージ生成時（LLMコール#1）に全段階を事前生成する
- ユーザーがヒントを要求した時点でのリアルタイムLLMコールは不要

---

## フェーズ/ブロックごとのヒント利用可否

| 場面 | 利用可否 |
|------|---------|
| フェーズA（初回発話） | 全段階利用可 |
| フェーズB（即時リトライ） | ヒント1のみ |
| フェーズC 本番発話 | ヒントなし |
| ブロック1（復習） | 全段階利用可 |
| ブロック3（本日の定着） | ヒント1のみ |

---

## SRSへの影響（信頼度乗数）

ヒント使用状況に応じて、SRSの間隔計算に信頼度乗数を適用する。

| ヒント使用 | 信頼度乗数 |
|-----------|-----------|
| ヒントなし | 1.0 |
| ヒント1 | 0.7 |
| ヒント2 | 0.4 |

### ヒント使用パターンの分析

- ヒント使用回数を記録し、SRSの信頼度乗数として適用する
- LLM層でもヒント使用パターンを分析する
  - 例: 「この項目は毎回ヒントを使う」と判断された場合、追加の練習機会を生成する
