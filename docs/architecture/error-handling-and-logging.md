# エラーハンドリング・ロギング方針

作成日: 2026-04-10

---

## エラーハンドリング

当面は単一ユーザー（開発者本人）のみ使用するため、外部サービス障害時の詳細なリカバリー設計は行わない。

- 各Adapterでtenacityによる指数バックオフリトライを設定する
- リトライ上限超過時は例外を伝播させ、エラーを表示する
- ユーザー向けの障害対応UX（部分スキップ、graceful degradation等）は必要になった時点で設計する

## ロギング

structlogを使用し、構造化ログを出力する。

### 方針

- ドメインコードにログ文をべた書きしない（architecture overview 9.3節）
- EventBus の emit 処理と Port の呼び出し境界で自動記録する
- JSONL形式で出力し、grep/jq でフィルタ可能にする

### 記録対象

- イベントの発火・ハンドラ実行（Event Bus が自動記録）
- 外部APIコールの入出力・レイテンシ・成否（Adapter層で記録）
- エラー発生時のコンテキスト（tenacityリトライ含む）
